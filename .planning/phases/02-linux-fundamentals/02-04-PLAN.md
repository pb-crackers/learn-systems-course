## Overview

Brief intro: "This is your quick reference for all Linux commands and concepts covered in this module. Bookmark this page — you'll come back to it often."

Then use multiple QuickReference components, one per lesson topic. Each section should include the most important commands from that lesson (not every command — the essentials a learner would look up).

**Section 1: Hardware & System Info** (from LNX-01)
- lscpu — Display CPU architecture information — `lscpu`
- free -h — Show memory usage in human-readable format — `free -h`
- lsblk — List block devices (disks and partitions) — `lsblk`
- df -h — Show filesystem disk space usage — `df -h`
- cat /proc/cpuinfo — Detailed CPU information from kernel
- cat /proc/meminfo — Detailed memory information from kernel

**Section 2: Kernel & OS** (from LNX-02)
- uname -a — Print system information (kernel version, hostname, arch) — `uname -a`
- strace — Trace system calls of a command — `strace ls`
- dmesg — Print kernel ring buffer messages — `dmesg | tail -20`
- /proc/[pid]/ — Per-process information directory — `ls /proc/1/`

**Section 3: Filesystem** (from LNX-03)
- ls -lai — List files with inode numbers and details — `ls -lai /etc`
- stat — Display file/directory detailed status (inode, permissions, times) — `stat /etc/passwd`
- ln — Create hard link — `ln original.txt hardlink.txt`
- ln -s — Create symbolic link — `ln -s /path/to/target symlink`
- mount / findmnt — Show mounted filesystems — `findmnt --real`
- df -i — Show inode usage — `df -i`
- du -sh — Show directory disk usage — `du -sh /var/log`

**Section 4: Permissions** (from LNX-04)
- chmod — Change file permissions (numeric) — `chmod 755 script.sh`
- chmod — Change file permissions (symbolic) — `chmod u+x,g-w file`
- chown — Change file owner — `chown user:group file`
- umask — Display/set default permission mask — `umask 022`
- id — Display current user and group IDs — `id`
- whoami — Print current username — `whoami`

**Section 5: Processes** (from LNX-05)
- ps aux — List all running processes — `ps aux | grep nginx`
- top — Real-time process monitor — `top`
- kill — Send signal to process — `kill -15 PID` (SIGTERM)
- kill -9 — Force kill process — `kill -9 PID` (SIGKILL)
- kill -l — List all signal names and numbers
- jobs / fg / bg — Manage background processes — `jobs`, `fg %1`
- /proc/PID/cmdline — Read process command line

**Section 6: Shell** (from LNX-06)
- echo $PATH — Display PATH variable — `echo $PATH`
- export VAR=value — Set environment variable — `export DB_HOST=localhost`
- which / type — Find command location — `which python`, `type cd`
- cmd1 | cmd2 — Pipe stdout to stdin — `ps aux | grep node`
- cmd > file — Redirect stdout to file (overwrite) — `ls > files.txt`
- cmd >> file — Redirect stdout to file (append) — `echo "line" >> log.txt`
- cmd 2>&1 — Merge stderr into stdout — `cmd > out.txt 2>&1`
- $? — Exit code of last command — `echo $?`

**Section 7: Scripting** (from LNX-07)
- #!/usr/bin/env bash — Portable shebang line
- set -euo pipefail — Strict error handling (exit on error, unset vars, pipe fail)
- if [ condition ]; then ... fi — Conditional execution
- for item in list; do ... done — Loop over items
- function name() { ... } — Function definition
- trap 'cleanup' EXIT — Run cleanup on exit
- $1, $2, $#, "$@" — Script arguments (first, second, count, all)

**Section 8: Text Processing** (from LNX-08)
- grep pattern file — Search for pattern in file — `grep "ERROR" app.log`
- grep -r — Recursive search — `grep -r "TODO" src/`
- sed 's/old/new/g' — Replace text globally — `sed 's/http/https/g' urls.txt`
- awk '{print $1}' — Print first field — `awk '{print $1}' access.log`
- sort — Sort lines — `sort -rn` (reverse numeric)
- uniq -c — Count adjacent duplicates (sort first!) — `sort | uniq -c`
- cut -d: -f1 — Extract field by delimiter — `cut -d: -f1 /etc/passwd`
- xargs — Convert stdin to arguments — `find . -name "*.tmp" | xargs rm`

**Section 9: Package Management** (from LNX-09)
- apt-get update — Refresh package index (always run first!)
- apt-get install — Install package — `apt-get install -y curl`
- apt-get remove — Remove package (keep config) — `apt-get remove nginx`
- apt-get purge — Remove package and config — `apt-get purge nginx`
- apt-cache search — Search available packages — `apt-cache search json`
- apt-cache show — Show package details — `apt-cache show curl`
- dpkg -l — List installed packages — `dpkg -l | grep nginx`
- dpkg -L — List files in installed package — `dpkg -L curl`

Each section should be a separate `<QuickReference>` component with a descriptive title matching the lesson topic.
