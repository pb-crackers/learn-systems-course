verify.sh PASS/FAIL pattern:
```bash
#!/usr/bin/env bash
set -euo pipefail
PASS=0
FAIL=0
check() {
  local desc="$1"
  local result="$2"
  if [ "$result" = "pass" ]; then
    printf "  \033[32mPASS\033[0m: %s\n" "$desc"
    PASS=$((PASS + 1))
  else
    printf "  \033[31mFAIL\033[0m: %s\n" "$desc"
    FAIL=$((FAIL + 1))
  fi
}
# checks here...
echo ""
if [ "$FAIL" -eq 0 ]; then
  printf "\033[32mRESULT: PASS\033[0m — All %d checks passed.\n" "$PASS"
else
  printf "\033[31mRESULT: FAIL\033[0m — %d of %d checks failed.\n" "$FAIL" "$((PASS + FAIL))"
  exit 1
fi
```

IMPORTANT: Use `$((PASS + 1))` arithmetic, NOT `((PASS++))` — the latter exits with status 1 when result is 0 under set -e.

**Dockerfile** at `docker/linux/Dockerfile`:
```dockerfile
FROM ubuntu:22.04
ENV DEBIAN_FRONTEND=noninteractive
RUN apt-get update && apt-get install -y \
    procps \
    gawk \
    apt-utils \
    curl wget vim nano less tree \
    strace \
    man-db \
    iproute2 \
    && rm -rf /var/lib/apt/lists/*
COPY verify/ /usr/local/lib/learn-systems/verify/
COPY setup/ /usr/local/lib/learn-systems/setup/
RUN chmod +x /usr/local/lib/learn-systems/verify/*.sh \
             /usr/local/lib/learn-systems/setup/*.sh
RUN useradd -m -s /bin/bash student && \
    useradd -m -s /bin/bash testuser && \
    groupadd devops && \
    usermod -aG devops student
WORKDIR /home/student
CMD ["/bin/bash"]
```

Note: `strace` for LNX-02 OS lesson, `man-db` for general reference, `iproute2` for `ss` (used in later modules but available here). The `student` and `testuser` accounts enable the permissions exercise (LNX-04). The `devops` group is for group permission exercises.

**Verify script 01-computers.sh**: Check that the learner can answer questions by creating answer files:

- check: `/tmp/exercise/total-memory.txt` exists (learner wrote total memory from /proc/meminfo)
- check: `/tmp/exercise/block-devices.txt` exists (learner wrote output of lsblk)

**Setup script 02-os.sh**: Create /tmp/exercise/ directory. Print instructions to explore /proc, use strace on a simple command, examine /proc/1/.

**Verify script 02-os.sh**: Check:
- `/tmp/exercise/kernel-version.txt` exists (learner wrote output of uname -r)
- `/tmp/exercise/strace-output.txt` exists and contains "write(" or "openat(" (learner ran strace and saved output)
- `/tmp/exercise/init-cmdline.txt` exists (learner read /proc/1/cmdline)

**Setup script 03-filesystem.sh**: Create /tmp/exercise/ with subdirectories. Create some sample files for inode exploration. Create a file to practice hard/soft links on.

**Verify script 03-filesystem.sh**: Check:
- A hard link exists (two files with same inode number): `[ "$(stat -c %i /tmp/exercise/original.txt)" = "$(stat -c %i /tmp/exercise/hardlink.txt)" ]`
- A symbolic link exists: `[ -L /tmp/exercise/softlink.txt ]`
- `/tmp/exercise/mount-info.txt` exists (learner captured mount or findmnt output)
- Learner listed inodes: `/tmp/exercise/inode-listing.txt` exists

**Setup script 04-permissions.sh**: Create /tmp/exercise/ with files that need permission changes: secret.txt (currently world-readable, needs 600), a scripts/ directory, a shared/ directory for sticky bit practice. Create as root, then switch context instructions.

**Verify script 04-permissions.sh**: Check:
- `[ "$(stat -c '%a' /tmp/exercise/secret.txt)" = "600" ]` — secret.txt is owner-read-write only
- `[ -x /tmp/exercise/scripts ]` — scripts/ directory is executable
- `[ "$(stat -c '%a' /tmp/exercise/shared)" = "1777" ]` or check sticky bit: `[ -k /tmp/exercise/shared ]`
- `/tmp/exercise/scripts/deploy.sh` is executable by owner

**Setup script 05-processes.sh**: Create /tmp/exercise/. Launch a background process (a simple sleep loop or a busy-wait script) that the learner needs to find and manage. Create a script that will produce a zombie process when run.

**Verify script 05-processes.sh**: Check:
- `/tmp/exercise/found-pid.txt` exists (learner found the PID of the background process)
- `/tmp/exercise/process-cmdline.txt` exists (learner read /proc/PID/cmdline)
- `/tmp/exercise/signal-sent.txt` exists (learner confirms they sent a signal)
