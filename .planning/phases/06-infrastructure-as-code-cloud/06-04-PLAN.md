**docker/cloud/mapping-exercise/compose.yml (CLD-06)**
Based on RESEARCH.md Pattern 4 but expanded to the recommended 4-service pattern (loadbalancer + app1 + app2 + persistent volume). Include inline comments mapping each component to cloud services:

```yaml
# This Compose stack mirrors a cloud architecture.
# Each component maps to a cloud service — the lesson teaches the mapping.
services:
  loadbalancer:          # Cloud: ALB (AWS) / Cloud Load Balancing (GCP) / Azure Load Balancer
    image: nginx:1.25
    ports:
      - "8080:80"
    volumes:
      - ./nginx.conf:/etc/nginx/nginx.conf:ro
    depends_on:
      - app1
      - app2

  app1:                  # Cloud: EC2/ECS task (AWS) / GCE/Cloud Run (GCP) / Azure VM/Container
    image: nginx:1.25    # Represents "AZ-1 replica"
    expose:
      - "80"

  app2:                  # Cloud: EC2/ECS task (AWS) / GCE/Cloud Run (GCP) / Azure VM/Container
    image: nginx:1.25    # Represents "AZ-2 replica"
    expose:
      - "80"

volumes:
  app-data:              # Cloud: EBS (AWS) / Persistent Disk (GCP) / Azure Disk Storage
    driver: local

networks:
  default:               # Cloud: VPC with private subnet
    driver: bridge
```

**docker/cloud/mapping-exercise/nginx.conf**
Minimal nginx config that load-balances between app1 and app2:
```nginx
events {}
http {
  upstream app_servers {
    server app1:80;
    server app2:80;
  }
  server {
    listen 80;
    location / {
      proxy_pass http://app_servers;
    }
  }
}
```

Frontmatter: title "Cloud Fundamentals Cheat Sheet", module "Cloud Fundamentals", moduleSlug "07-cloud", lessonSlug "06-cheat-sheet", order 6, difficulty "Foundation", estimatedMinutes 5, prerequisites [], tags ["cloud", "cheat-sheet", "aws", "gcp", "azure", "reference"].

Create 5 QuickReference sections (one per lesson topic):

1. "Cloud Concepts" — items: IaaS definition (you manage OS+app, provider manages hardware), PaaS definition (you manage app, provider manages OS+hardware), SaaS definition (provider manages everything), regions (physical locations), availability zones (isolated within region), multi-AZ for high availability.

2. "Compute Services" — items: VMs (EC2/Compute Engine/Azure VMs — full OS, SSH access), Managed Containers (ECS-Fargate/Cloud Run/Container Apps — run Docker images without managing hosts), Serverless (Lambda/Cloud Functions/Azure Functions — per-invocation, event-driven), when to use each tier.

3. "Cloud Networking" — items: VPC (isolated network = docker network), Subnets (public=internet-reachable, private=internal-only), Load Balancer (ALB/Cloud LB/Azure LB = nginx proxy), Security Groups (stateful firewall rules = iptables), DNS (Route 53/Cloud DNS/Azure DNS).

4. "Cloud Storage" — items: Block Storage (EBS/Persistent Disk/Azure Disk = docker volume — attached to one instance), Object Storage (S3/Cloud Storage/Azure Blob — HTTP-accessible, no filesystem), File Storage (EFS/Filestore/Azure Files — NFS shared mount), when to use each type.

5. "IAM" — items: Users/Groups (= Linux users/groups), Policies (= chmod/chown — who can do what on which resource), Roles (= sudo — temporary privilege escalation), Service Accounts/Managed Identity (identity for applications, not humans), Least Privilege (start with no access, add only what is needed).

Include a "What's Next" Callout at the end bridging to Phase 7: "You now understand infrastructure as code and cloud concepts. In the next module, you will learn to monitor all of this with Prometheus and Grafana — turning running infrastructure into observable systems."
