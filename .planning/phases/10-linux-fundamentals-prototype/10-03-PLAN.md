**06-shell-fundamentals.mdx** — "Diagnose a Broken PATH and Build a Pipeline" scenario: deployment container failing with 'command not found'.
- Question examples: "After restoring PATH, the container can run ls again. But the deployment still fails because a custom tool at /opt/tools/bin/deploy is not found. What PATH modification would fix this, and where would you put it so it persists?" / "Your monitoring pipeline ps aux | grep bash | awk prints the PIDs. If you need to pass these PIDs to kill, which tool bridges the gap between pipe output and command arguments?"

**07-shell-scripting.mdx** — "Write a Deployment Backup Script" scenario: logistics company, manual config copy with no backup or validation.
- Question examples: "Your script uses set -euo pipefail. A colleague's script does not, and it once deleted the production config because BACKUP_DIR was unset. Which flag in set -euo pipefail would have caught this, and what would have happened?" / "The backup function copies config with a timestamp suffix. If the script fails halfway through deployment, the trap cleanup runs. Why is trap EXIT more reliable than putting cleanup at the end of the script?"

**08-text-processing.mdx** — "Analyze a Web Server Access Log" scenario: SRE at e-commerce company, elevated error rates, 50k-line log.
- Question examples: "You built a pipeline showing /api/users/search has the most 500 errors. The team asks which client IP is hitting that endpoint most. How would you modify your pipeline to answer this?" / "Your colleague ran sort on the access log then uniq -c, but got wrong counts because they forgot a step. What did they forget, and why does uniq only work on adjacent lines?"
