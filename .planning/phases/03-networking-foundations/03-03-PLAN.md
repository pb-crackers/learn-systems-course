**docker/networking/Dockerfile** — Base networking image used as `client` in all Compose labs:
```dockerfile
FROM ubuntu:22.04
ENV DEBIAN_FRONTEND=noninteractive
RUN apt-get update && apt-get install -y \
    # DNS tools
    dnsutils \
    # HTTP tools
    curl wget \
    # Network inspection
    iproute2 \
    iputils-ping \
    traceroute \
    # Port checking
    netcat-openbsd \
    # Packet capture
    tcpdump \
    # SSH client
    openssh-client \
    # Firewall tools
    iptables \
    ufw \
    # General
    vim nano less \
    && rm -rf /var/lib/apt/lists/*
WORKDIR /tmp/exercise
CMD ["sleep", "infinity"]
```

**docker/networking/02-tcp-ip/compose.yml** — Two-container lab on a custom bridge network with static IPs so learners can inspect specific IP/subnet values:
```yaml
services:
  server:
    build: ..
    networks:
      lab-net:
        ipv4_address: 10.0.0.10
    command: sleep infinity
  client:
    build: ..
    volumes:
      - ./setup.sh:/setup.sh:ro
      - ./verify.sh:/verify/02-tcp-ip.sh:ro
    networks:
      lab-net:
        ipv4_address: 10.0.0.20
    command: sleep infinity
networks:
  lab-net:
    driver: bridge
    ipam:
      config:
        - subnet: 10.0.0.0/24
```
Note: `build: ..` references the parent Dockerfile. Static IPs used here because the lesson specifically teaches IP addressing and subnet inspection.

**docker/networking/02-tcp-ip/setup.sh** — Creates /tmp/exercise directory, prints INSTRUCTIONS telling learner to inspect IPs, routes, and ports using ip addr, ip route, ss commands.

**docker/networking/02-tcp-ip/verify.sh** — Checks:
1. Learner saved IP address info to /tmp/exercise/ip-info.txt
2. Learner saved routing table to /tmp/exercise/routes.txt

**docker/networking/05-ssh/Dockerfile.sshd** — Custom SSH server image:
```dockerfile
FROM ubuntu:22.04
ENV DEBIAN_FRONTEND=noninteractive
RUN apt-get update && apt-get install -y openssh-server && \
    rm -rf /var/lib/apt/lists/* && \
    mkdir /run/sshd && \
    useradd -m -s /bin/bash student && \
    echo "student:changeme" | chpasswd
```
