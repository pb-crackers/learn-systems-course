Explain the feedback loop: fast CI catches bugs in minutes instead of days. Show a simple pipeline diagram using ASCII:

```
       [Notify]  [Notify]
```

**04-deployment-strategies.mdx (CICD-04)**
Frontmatter: title "Deployment Strategies: Blue/Green, Rolling, and Canary", module "CI/CD Pipelines", moduleSlug "05-cicd", lessonSlug "04-deployment-strategies", order 4, difficulty "Intermediate", estimatedMinutes 20, prerequisites ["05-cicd/03-building-testing"], tags ["deployment", "blue-green", "rolling", "canary", "rollback", "zero-downtime", "strategies"].

How It Works section: Explain the problem — deploying new code means replacing running processes, and naive deployment (stop old, start new) causes downtime. Explain three strategies conceptually (per locked decision — diagrams only, no implementation):

**Blue/Green Deployment:**
Two identical environments. Blue = current live, Green = new version idle. Deploy to Green, test, then switch the load balancer. Instant rollback by switching back to Blue. ASCII diagram from RESEARCH.md. Tradeoffs: doubles infrastructure cost, instant rollback, but database migrations complicate the picture.

**Rolling Deployment:**
Replace instances one at a time (or in batches). At any point, both old and new versions are running simultaneously. ASCII diagram from RESEARCH.md. Tradeoffs: no extra infrastructure, but rollback is slow (must re-deploy old version), and both versions must be compatible with the same database schema.

**Canary Deployment:**
Route a small percentage of traffic (1-5%) to the new version. Monitor error rates and latency. If metrics are healthy, gradually increase traffic. ASCII diagram from RESEARCH.md. Tradeoffs: safest for detecting issues, but requires traffic splitting infrastructure (load balancer rules, service mesh), and monitoring is essential (connects to Phase 7).

Callout type="info": "You won't implement these strategies hands-on in this course — they require production load balancers and traffic management. But understanding these concepts is critical for any DevOps role. When you see 'blue/green deployment' in a job description, you will know exactly what it means."

Compare the three strategies in a summary table: downtime risk, infrastructure cost, rollback speed, complexity.

Exercise: ExerciseCard difficulty="Intermediate", scenario "Your CTO asks which deployment strategy to use for a new e-commerce site. The site handles payments, so any bug in production is costly." Steps: analyze the tradeoffs (payment processing = high risk = canary makes sense), explain why blue/green is the fallback (instant rollback), explain why rolling is risky here (both versions handling payments during transition). VerificationChecklist: can explain each strategy, can recommend one with justification, understands rollback implications.
