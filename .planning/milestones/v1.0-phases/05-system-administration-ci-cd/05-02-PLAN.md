Add a Callout tip for Apple Silicon in Dockerfile.systemd comments: --platform linux/amd64 flag if image fails on ARM Macs.

**Setup scripts** (docker/sysadmin/setup/):
Each creates the exercise environment. Follow established pattern from docker/linux/setup/.

01-users.sh: Create testuser1/testuser2 accounts, create devteam/ops groups, pre-create /etc/sudoers.d/ directory. Leave exercise tasks for learner.
02-systemd.sh: Create /usr/local/bin/hello.sh (simple loop script that echoes timestamp every 5s). Leave unit file creation for learner.
03-logging.sh: Create /var/log/myapp/ directory, generate sample log entries with logger command, ensure rsyslog is running. Leave logrotate config for learner.
04-disk.sh: Create loopback file with truncate -s 512M /tmp/disk.img and losetup /dev/loop0 /tmp/disk.img. Leave partitioning/formatting/mounting for learner.
05-scheduling.sh: Create /usr/local/bin/backup.sh and /usr/local/bin/report.sh scripts. Ensure cron daemon is running. Leave crontab and timer creation for learner.
06-monitoring.sh: Install stress or use bash CPU/IO generators to create artificial load scenarios. Create 3 scenarios: cpu-bound (tight loop), io-bound (dd to /dev/null), memory-pressure (allocate large arrays). Leave identification for learner.

**Verify scripts** (docker/sysadmin/verify/):
Each uses the established PASS/FAIL pattern from docker/linux/verify/ — set -euo pipefail, check() function, PASS/FAIL counters, final RESULT line.

01-users.sh: Check user exists (id username), check group membership (groups username | grep groupname), check sudoers config (sudo -l -U username).
02-systemd.sh: Check unit file exists (/etc/systemd/system/hello.service), check service is active (systemctl is-active hello), check service is enabled (systemctl is-enabled hello), check journalctl has output (journalctl -u hello --no-pager | grep -q "").
03-logging.sh: Check logrotate config exists (/etc/logrotate.d/myapp), check rotation works (logrotate -d), check journalctl filtering (journalctl -u rsyslog --since "1 hour ago").
04-disk.sh: Check loopback is mounted (mount | grep /mnt/data), check filesystem exists (df -h /mnt/data), check fstab entry (grep /mnt/data /etc/fstab).
05-scheduling.sh: Check crontab entry (crontab -l | grep backup), check timer unit active (systemctl is-active backup.timer 2>/dev/null || true — may not exist in standard container, check conditionally).
06-monitoring.sh: Check learner can identify load average (uptime output captured), check vmstat ran (check for vmstat output file or just validate tool is installed).
