**Step 2: Add mode resolution function**

Add a `resolveMode` function inside the component (or above it). This implements the exact contract from foundation-safety-net.md:

```typescript
function resolveMode(
  difficulty: Difficulty,
  modeProp: ExerciseMode | undefined,
  preferredMode: ExerciseMode | null
): ExerciseMode {
  // Foundation safety net — hard override, skip ALL other checks
  if (difficulty === 'Foundation') {
    return 'guided'
  }

  // Priority 1: explicit mode prop
  if (modeProp) {
    return modeProp
  }

  // Priority 2: learner preference
  if (preferredMode) {
    // 'compose' is not valid for Intermediate — fall back to default
    if (difficulty === 'Intermediate' && preferredMode === 'compose') {
      return DIFFICULTY_MODE_DEFAULT[difficulty]
    }
    return preferredMode
  }

  // Priority 3: difficulty default
  return DIFFICULTY_MODE_DEFAULT[difficulty]
}
```

**Step 3: Use resolveMode in the component**

Inside the component function:
```typescript
const { preferredMode } = useProgress()
const effectiveMode = resolveMode(difficulty, mode, preferredMode)
```

**Step 4: Implement three render branches for the expanded content area**

Replace the existing steps rendering in the expanded section with mode-branched rendering:

**Guided mode (`effectiveMode === 'guided'`):**
- Show objective (same as current)
- Show numbered steps list (same as current structure)
- For each step: show step number, description, command (if present)
- NEW: If `annotated === true` AND the step has `annotations` array AND `annotations.length > 0`, render `<AnnotatedCommand command={s.command!} annotations={s.annotations} />` INSTEAD of the plain `<code>` block for that step's command.
