<Callout type="tip" title="What's Next: Docker Networking">
Everything you learned in this module — bridge networks, container DNS, IP routing, port
mapping, firewall rules — is exactly what Docker networking builds on. When you run
`docker network create mynet`, Docker creates a Linux bridge interface. When containers
resolve each other by service name, that's the same DNS mechanism you explored in the DNS
lesson. When you publish a port with `-p 8080:80`, that's iptables NAT rules under the
hood. The Docker module will feel familiar because you already understand the primitives.
</Callout>
