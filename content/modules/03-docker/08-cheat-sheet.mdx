## Dockerfile Best Practices

<QuickReference
  sections={[
    {
      title: "Dockerfile Best Practices (Lesson 7)",
      items: [
        { command: "FROM <image> AS <name>", description: "Multi-stage: name a build stage for reference in later COPY --from instructions", example: "FROM node:20-alpine AS builder" },
        { command: "COPY --from=<stage> <src> <dst>", description: "Multi-stage: copy only the output of a prior stage — intermediate build tools are excluded from the final image", example: "COPY --from=builder /app ." },
        { command: "COPY package*.json ./", description: "Cache-friendly order: copy dependency manifests before source — npm install is cached unless packages change" },
        { command: "RUN npm ci --omit=dev", description: "Reproducible install from package-lock.json — faster and safer than npm install in CI/CD" },
        { command: "node_modules", description: ".dockerignore entry: exclude local node_modules from build context — container installs its own" },
        { command: ".git", description: ".dockerignore entry: exclude git history from build context — never needed in the image" },
        { command: ".env", description: ".dockerignore entry: CRITICAL — never send secrets to the build context" },
        { command: "RUN addgroup -S appgroup && adduser -S appuser -G appgroup", description: "Create a non-root system user — all subsequent instructions including CMD run as this user" },
        { command: "USER appuser", description: "Switch to non-root user — prevents container breakout from granting host root access" },
