## Overview

All monitoring concepts, PromQL patterns, Grafana configurations, and incident response templates from this module in one place. Use this as a reference when you are writing queries, building dashboards, or responding to incidents.

## Incident Response

<QuickReference
  sections={[
    {
      title: "Incident Response (Lesson 5)",
      items: [
        { command: "Alert Lifecycle", description: "1. Metric exceeds threshold (HighRequestLatency expr evaluates true). 2. PENDING: alert fires on each evaluation but has not held for the 'for' duration yet. 3. FIRING: 'for' duration elapsed — alert sent to Alertmanager. 4. Alertmanager routes, groups, and deduplicates. 5. Notification sent to receiver (webhook, Slack, PagerDuty). 6.         { command: "Runbook format", description: "Symptoms: what the alert looks like, what users report. Escalation: who to page if resolution steps fail.", example: "HighMemoryUsage runbook:\nSymptoms: process_resident_memory_bytes growing\nDiagnosis: rate(nodejs_heap_used_bytes[5m]) > 0\nResolution: restart service; check for leak in code" },
        { command: "Postmortem format", description: "Summary: 1-2 sentences on what happened. Timeline: timestamped sequence from first symptom to resolution. Root Cause: the actual technical failure (not 'human error').
